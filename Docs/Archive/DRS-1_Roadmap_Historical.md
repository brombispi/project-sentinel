# DRS-1 Roadmap

## v0.0.1 – Project Created
Status: Complete

- Project folder created.
- Documentation structure created.
- Vision document started.

## v0.1.0 – Basic Recovery Environment
Status: Planned

Goal: Prepare the Raspberry Pi 5 as a dedicated Linux recovery station.

Planned features:
- Confirm operating system.
- Enable SSH access.
- Install core recovery tools.
- Confirm external drive detection.
- Confirm SMART access.

## v0.2.0 – Manual Imaging Workflow
Status: Planned

Goal: Create a safe manual imaging workflow using ddrescue.

Planned features:
- Identify source drive.
- Identify destination drive.
- Create case folder.
- Run SMART report.
- Run ddrescue.
- Save logs.

## v0.3.0 – Case Folder Automation
Status: Planned

Goal: Automatically create structured case folders.

Planned features:
- Case number input.
- Automatic folder creation.
- SMART report saved automatically.
- Imaging log saved automatically.

## v0.5.0 – First Usable DRS-1 Prototype
Status: Planned

Goal: Process a test drive from start to finish using DRS-1.

Planned features:
- Detect drive.
- Run SMART.
- Create image.
- Verify logs.
- Transfer image to Mac mini.

## v1.0.0 – Production Workflow
Status: Planned

Goal: Use DRS-1 confidently for real DigiRettung cases.

Planned features:
- Stable imaging workflow.
- Reliable logs.
- Clear case structure.
- Basic operator documentation.
