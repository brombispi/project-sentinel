Project Sentinel Engineering Values

Version: 0.1
Status: Active

Mission Critical Rule

Never perform an operation that increases the risk to the customer’s original data unless there is no safer alternative.

Engineering Values

1. Protect the original.
2. Image before recovery whenever possible.
3. Every important action must be logged.
4. Every diagnostic should be repeatable.
5. Every recommendation should be explainable.
6. Every recovery should be documented.
7. Simplicity is preferred over unnecessary complexity.
8. Reliability is more important than speed.
9. The operator should always know what Project Sentinel is doing.
10. If Project Sentinel is uncertain, it should stop and ask rather than guess.
